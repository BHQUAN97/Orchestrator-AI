# Spec Mode — Template

## Output format cho moi spec:

### 1. Tong quan
- Ten feature
- Muc dich
- Nguoi dung muc tieu

### 2. User Stories
- As a [role], I want [feature], so that [benefit]
- Viet tieng Viet co dau

### 3. Acceptance Criteria
- Given [context], When [action], Then [result]
- Bao gom ca happy path va edge cases

### 4. Technical Design
- API endpoints (neu co)
- Database changes (neu co)
- Component structure (neu co)
- Dependencies

### 5. Risks & Notes
- Security considerations
- Performance impact
- Breaking changes
- Migration needed?

### 6. Task Breakdown
- Chia thanh tasks 1-4 gio
- Uu tien: Critical → High → Medium → Low
- Goi y model: complex → smart, simple → default/cheap
